# Modifier le contenu d'une page du site

Page ou section à modifier : $ARGUMENTS

## Workflow

1. Lire `docs/ghostwriter.md` + `docs/copywriter.md`
2. Lire `docs/business.md`
3. Si la page contient des cas clients : lire `docs/missions.md` (missions concernées uniquement)
4. Respecter les contraintes Next.js SSG (pas de server-side, `trailingSlash: false`)
5. Proposer les modifications pour validation avant d'écrire

- Voix produit (pas "je") sauf page À propos et verbatim validé home page
- Pain d'abord, solution ensuite
- Titres outcome-based, prix affiché directement
- Témoignages avec résultat concret
