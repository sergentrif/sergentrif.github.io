Le SaaS : on build une fois, on déploie mille fois. Le coût marginal d'un utilisateur supplémentaire tend vers zéro. C'est le modèle.

Avec l'IA dans le produit, cette logique s'inverse. On build mille fois pour déployer mille fois. La consommation de tokens augmente avec l'usage. Les économies d'échelle n'existent pas.

Sur Timescanner, j'ai un temps envisagé d'utiliser l'IA pour calculer les temps de travail depuis le fichier agenda. C'est possible. Mais faire lire un fichier de 5Mo à un LLM à chaque requête HTTP, alors qu'un script le lit à coût nul, c'est sortir le bazooka pour tuer une mouche.

Ce n'est pas un argument contre l'IA dans un produit. C'est un argument pour comprendre ce qu'on décide vraiment quand on intègre un LLM dans une feature.

La question n'est pas "est-ce qu'on peut". C'est "est-ce que l'IA est le bon outil ici".
